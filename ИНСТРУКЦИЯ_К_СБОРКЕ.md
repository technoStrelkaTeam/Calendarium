# ИНСТРУКЦИЯ К СБОРКЕ СЕРВЕРА

### I. Создайте venv (желательно использовать PyCharm). 
###### если вы знаете как это сделать, то можете пропустить первый пункт
<img width="512" height="229" alt="image" src="https://github.com/user-attachments/assets/aa4da41f-20e0-49d3-bc5f-eb9b17508148" />

1. Для этого можно открыть любой .py файл и когда PyCharm предложит настроить интерпретатор нажмите Configure Python Interpreter
2. Add new interpreter -> add local interpreter...
3. Выберите Virtualenv Environment. Укажите путь (лучше всего Calendarium/backend/venv) и нажмите OK. Потом дождитесь создания venv
   <img width="919" height="536" alt="image" src="https://github.com/user-attachments/assets/23e8fa68-e701-421b-b17f-a2ad23020183" />


### II. Установите зависимости
1. В терминале перейдите в Calendarium/backend и пропишите команды ниже
2. pip install -r requirements.txt

### III. Установите OLLAMA для встроенной в проект нейросети и запустите его
1. Скачайте Ollama с официального сайта https://ollama.com/download , либо в терминале:
   
   На Windows: irm https://ollama.com/install.ps1 | iex
   
   На Linux или macOS: curl -fsSL https://ollama.com/install.sh | sh
3. В терминале выполните команды
   
   ollama serve
   
   ollama pull gemma3:1b


P.S. Если вам позволяют ресурсы компьютера, то можете взять более хорошую модель, либо вовсе зарегестрироваться на Ollama и взять облачную. Тогда в ollama pull вместо gemma3:1b пишите любую другую модель из https://ollama.com/search и в файле Calendarium/backend/api/config.py в константе AI_MODEL заменяете имя модели на свою.


### IV. Запустите проект
1. В терминале перейдите в папку Calendarium/backend и выполните команду

   uvicorn api.main:app --reload  --host 0.0.0.0








# ИНСТРУКЦИЯ ПО СБОРКЕ КЛИЕНТА

### I. Установите Flutter и зависимости
Подробнее: https://docs.flutter.dev/install
1. Используйте Visual Studio Code. Откройте папку Calendarium/client и на запрос от Visual Studio Code установите Flutter.
2. Введите команду в терминал в той же папке
   flutter pub get

### II. Сборка

1. В зависимости от желанной платформы выполните команду

   
   WEB: flutter build web



   Android: flutter build apk --release



   *(!!! УБЕДИТЕСЬ, ЧТО ПО ПУТИ К ПРОЕКТУ ТОЛЬКО ЛАТИНСКИЕ СИМВОЛЫ !!!)*



   IOS: flutter build ios


   
После можете запустить этот файл на вашем устройстве.




В случае в вебом недостаточно просто запустить index.html. Выполните команды в терминале




   pip install http.client
   python --http.server 8080



   
   В папке с собранным вебов и перейдите по адресу localhost:8080

### III. Запуск без сборки (тест)
   В терминале в той же папке команда
   flutter run

Подключите android-устройство с режимом отладки перед выполнением команды если хотите тестировать android (с ios соответствующе)
